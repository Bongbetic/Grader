# Testability & Iteration-Readiness

A prompt is only as good as your ability to tell whether its output passed. Write the pass/fail check before you run the prompt, not after you see the result.

Iteration works when you change one variable at a time. If you rewrite everything at once, you cannot tell which change helped. Keep one clear format and test one knob per run.

- Try this: Write the check first: "Pass if the summary contains all three metrics and no invented numbers."
