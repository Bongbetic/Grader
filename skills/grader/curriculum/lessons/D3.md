# Specificity & Constraints

Scope boundaries turn an open request into a solvable one. Constraints include length, format, must-haves, must-nots, and any hard limits.

After adding constraints, read the prompt twice. If two instructions contradict each other, the model will obey the last one or hedge. Pick the rule that matters and drop the rest.
